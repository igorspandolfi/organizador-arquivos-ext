# Contribuindo

Obrigado por considerar contribuir! Aqui estão algumas diretrizes para ajudar.

## Como Reportar Bugs

- Use o GitHub Issues para reportar bugs.
- Descreva o maior número possível de detalhes.
- Forneça exemplos específicos para demonstrar os passos.
- Descreva o comportamento observado e o que você esperava.

## Como Sugerir Melhorias

- Use o GitHub Issues com o rótulo `enhancement` para sugestões.
- Explique claramente os casos de uso e os benefícios.

## Pull Requests

1. Fork o repositório.
2. Crie um branch (`git checkout -b feature/sua-feature`).
3. Commit suas mudanças (`git commit -am "Adiciona nova feature"`).
4. Push para o branch (`git push origin feature/sua-feature`).
5. Abra um Pull Request.

## Standards de Código

- Siga o PEP 8.
- Use type hints para funções.
- Adicione docstrings em português.
- Mantenha testes quando aplicável.

## Testes

Execute qualquer teste existente antes de submeter um PR:

```bash
python -m pytest
```
